伪春菜术语 

伺か 
日文读作ukagaka ，中文较常用之名称为「伪春菜」，另有以下日文旧称：「何か」、「何か。（）」、「任意」、「あれ以外の何か」。
伺か的整体架构如下：

核心程式已有专门的开发团队在进行开发，并提供我们所谓的「SAKURA Script」的语法规则，只要我们照着规则输入字串的话，就能显现出我们想要的动作。其中，「SHIORI」可视为伪AI的思考回路，「辞书」则扮演着伪AI的记忆这样一个角色。

核心程式(baseware) 
整个伺か的核心部位，启动人格时不可或缺的主程式，如SSP。
主要的工作在于：
告知人格有无事件发生。
执行 SAKURA Script 。
SHELL、BALLON、右键选单的显示。

GHOST (ゴ─スト, 人格) 
GHOST一词引用自攻壳机动队(Ghost in the Shell) 中所描述的GHOST定义，概括生命现象产生的个性、行为、记忆等人脑思绪统合的情报资讯，即俗称的灵魂。一个只有身体而没有灵魂的人，就等于没了生命。相对的，没有GHOST的SHELL，也只是一个空壳而已。
伪春菜便引继自这个概念，让使用者们自由制作属于自己的GHOST，然后与自制的或网路上公开的SHELL合为一体，并命名(NAMED)它们后，「她」便有了生命。
GHOST包含以下三种内容：
辞书(Dict)
伪AI(Shiori)
人格套件(Saori)

SHIORI (, 伪AI) 
SHIORI是执行、编译辞书成为可运作人格的程式依据，一般为一个DLL文件档。当核心程式接受到了指令，会将这个资讯传给Shiori，然后由Shiori从辞书文件中选出适当的反应来交由核心程式表现。不同的Shiori在辞书文件内使用的语法也会略有不同。
之所以会将Shiori与核心程式分开，一方面目的是提高伪春菜的自由性，另一方面也使系统开发者不必多去顾虑人工智慧开发之部份。故此举在当时亦可分担开发上的负担。
伪AI在人格的作用中扮演着大脑一般的角色，负责接收外来的资讯并读取辞书以传回定义好的反应与动作脚本。
「伪」，即假的意思，顾名思意即假的AI。由于伪春菜在运作上比较像是照着剧本去念台词，而非真实的理解其意义，故有此一词之说。

SAORI (さおり, 人格套件) 
GHOST追加机能用的插件(plug-in)，提供SHIORI所没有的功能。一般来说，只要是能直接套用到人格身上，符合Saori的标准规格并可用Shiori呼叫的特殊插件(plug-in)，便可称之为人格套件。
套件的目的在于追加伪AI中所没有提供的功能，诸如音乐播放、绘图、资料库、剪贴簿等来增进人格的性能，借以做出一般人格所做不到的事。

SHELL (シェル, 外形, 外表, 躯体) 
SHELL是一个伪春菜用来与使用者互动的一个介面，借此呈现角色外形、动作以及各式的反应。主要包含以下几种档案：
descript.txtShell的设定资讯。
surface.txt设定每个Surface所产生的图片、动画及滑鼠感应部位。
surfaceXXXX.png作为Surface根据的图片档，XXXX为任意数字。
surfaceXXXX.pna遮罩文件，作用于同档名的PNG图片上，会让遮罩之外的图片区块透明化。如果没有此档案的话，核心程式会把图片座标(0,0)的颜色视为透明色。
surfacetable.txt定义Surface之组别与名称，并显示在『表情测试』上。按Ctrl+T可显示。没有此档的话核心程式会套用预设的群组。
由于在中文中并没有能很好表达Shell一词完整定义的字词，往往会造成使用者的误会，故大多数人仍使用Shell这个字眼。

SURFACE (サ`フィス, 表情, 表相) 
在桌面上展现人格外形所使用的图片，是Shell的最基本要素。一般来说，surface0是使仆端(Sakura端)的标准样貌，surface10则是使魔端(Kero端)的标准样貌。
Surface的图片除了基本的表情样貌外，同时也包括动画所需的图片及各式各样的动作，藉由surface.txt文件进行图片的连结与连续动作。
表相：借一个有相的东西来表显一个没有形相的真理。

SAKURA Script (さくらScript, 樱语, 言灵) 
SAKURA Script是伪春菜间共通的一种指令描述脚本，并被所有的伪春菜核心程式所支援(依不同核心而有不同的支援度)。
核心程式会自行分析由人格的思考中枢所传出的对话脚本，将脚本中的SAKURA指令一一解析并表现出来。反过来说，人格的作者(创师)便是透过这些指令来操作核心程式的表相行为，借以展现人格的灵性。因此，SAKURA Script又可说是人格与核心程式间的沟通桥梁，是制作人格的人们所必须的基础知识。
常现的语法有：
\0 - 使仆端「说话」。
\1 - 使魔端「说话」。
\n - 换行。
\s[?] - 表情改变。\s[0]为使仆端预设表情，\s[10]为使魔端预设表情。
\w? - 对话延迟。\w8延迟0.4毫秒。

SSTP 
一种伺か专用的通讯协定，外部的应用程式可经由此协定传送SAKURA Script 给核心程式，如SSTP Bottle。

电脑术语 

API 
应用程序介面，（Application Programming Interface，简称API），就是软体系统不同组成部分衔接的约定。
应用程序介面为：「『电脑作业系统』或『程式函式库』提供给应用程式呼叫使用的原始码」。其主要目的是让应用程式开发人员得以呼叫一组常式功能，而无须考虑其底层的原始码为何、或理解其内部工作机制的细节。API本身是抽象的，它仅定义了一个介面，而不涉入应用程式如何实现的细节。

GUI 
图形使用者界面或图形用户介面(Graphical User Interface，GUI)是指采用图形方式显示的电脑操作环境用户介面(如Windows)。与早期电脑使用的命令行界面(如DOS)相比，图形界面对于用户来说更为简便易用。
